Cauldron 0.5.0

Multi-business platform — declarative DDL, RBAC, dynamic UI, CMS, knowledge base + LLM API

0.5.0

2026-03-15

Refactored

Core Framework RefactoringPhase 34

  • Moved 17 generic utilities from app layer to framework: things every Cauldron app needs now live in core
  • New cauldron.http-client module: http-post-json for outbound HTTP via curl subprocess
  • New cauldron.llm module: provider protocol (Claude, Ollama, Mock), tool system, conversation CRUD, agent-turn loop
  • cauldron.runtime:ht and ht-get — hash-table builder + polymorphic getter (works on hash-tables and alists)
  • API response helpers moved to cauldron.crucible: json-response, json-error, wrap-api-response, role-allows-p
  • Generic plugs moved to cauldron.crucible: plug-json-content-type, plug-html-content-type, redirect-with-flash
  • Module count: 15 → 17 | Test count: 1,446 → 1,487 (41 new tests)

Framework HardeningPhase 35

  • Centralized config: get-env, defconfig macro, validate-config, config-summary with secret masking
  • Request logging: plug-request-log + log-request-completion with method/path/status/duration
  • CORS middleware: make-plug-cors with preflight handling, origin matching, credentials, expose-headers
  • Rate limiting: make-plug-rate-limit per-IP tracking with sliding window, 429 + Retry-After
  • Response compression: pure-CL DEFLATE/gzip (RFC 1951/1952), compress-response plug with content-type awareness
  • Test count: 1,487 → 1,531 (44 new tests) | LOC: ~20,700 → ~21,200

User LifecyclePhase 36

  • Password reset flow: forgot-password → email with token → reset form → password update
  • Email verification: verification on signup, resend link, soft enforcement via dashboard banner
  • User settings page: profile editing, password change
  • Team invitations: admin invite → email → accept/signup flow
  • Test count: 1,531 → 1,584 (53 new tests) | LOC: ~21,200 → ~22,200

Canonical Logging + HTTP ClientPhase 37

  • New cauldron.logging module: Stripe-style canonical log lines
  • Enhanced HTTP client: GET/POST/PUT/PATCH/DELETE with timeout
  • Bridge integration: auto request_id generation, canonical log emission per request
  • Test count: 1,584 → 1,633 (49 new tests) | LOC: ~22,200 → ~22,600

Integration ProtocolPhase 38

  • New cauldron.integration module: declarative defintegration macro
  • Full AES-256-CBC encryption with NIST known-vector validation
  • Encrypted credential store with cascading scope resolution
  • Integration client with retry, rate limiting, health tracking
  • Webhook verification with HMAC-SHA256
  • Test count: 1,633 → 1,668 (35 new tests) | Modules: 18 → 19

Declarative Autonomous AgentsPhase 39

  • defagent macro: model, tools, triggers, memory, role in one form
  • Auto-tool derivation from Reagent resources and integration specs
  • Persistent agent memory with scope isolation
  • Multi-trigger system: PubSub, schedule, HTTP, webhook
  • Test count: 1,689 → 1,757 (68 new tests) | Modules: 19 → 20

Agent HardeningPhase 39B

  • Token budgets, A2A delegation with cycle detection, tool caching with TTL
  • SSE streaming, Forge agent dashboard, CLI agent commands
  • Test count: 1,757 → 1,799 (42 new tests) | Modules: 20 → 21

API Keys + Webhook Delivery + Billing + Uploads + Search + Activity + SecurityPhases 40–45

  • API key generation with SHA-256 hashing, scope-based access control
  • Webhook delivery with HMAC signing and exponential backoff
  • Subscription billing: state machine, metered usage, 402 gating
  • File uploads with UUID paths, MIME validation, polymorphic attachments
  • PostgreSQL full-text search: tsvector, GIN indexes, cross-table search
  • Activity feed: actor/action/target tracking with PubSub integration
  • Security: CSP, HSTS, IP allowlist, request size limits, session fixation protection
  • Test count: 1,799 → 1,970 (171 new tests)

Added

Application PlatformPhases 21–33

  • deftable macro: declarative table definitions generating CREATE TABLE DDL
  • RBAC: owner > admin > member > viewer hierarchy
  • Metadata-driven forms and tables — no hand-coded views per object
  • CMS template with Markdown rendering, CRM template, ecommerce, project management
  • Knowledge base + LLM integration with context building
  • Scry LiveView: client JS, real-time components, PubSub wiring
  • Self-documenting hypermedia API with _links and _actions
  • Static site generation with islands architecture
  • Schema-level object relations with cascading deletes
  • Dual agent runtime: platform (CLI) and tenant (web, RBAC-enforced)
  • Customer acquisition: domains, SEO, email, notifications, newsletters
